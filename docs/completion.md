# AI Completion

CodeX 1.0.0 treats code completion as an AI-first feature. The primary user experience is inline ghost text generated by the configured AI provider, enriched with editor context, nearby code, diagnostics, and related open files.

## Behavior

- AI ghost text triggers automatically after typing and can be requested manually with `Cmd/Ctrl + Shift + Space` or `Alt + \`.
- `Tab` accepts the visible AI suggestion.
- `Esc` dismisses the visible AI suggestion.
- LSP stays available for language intelligence such as diagnostics, hover, symbols, and definitions. It is not used as a local lexical fallback for AI completion.

Automatic completions use a fast path with compact prefix/suffix context and a small token budget. Manual completions use richer agentic context for larger edits or when more project awareness is useful.

## Provider Setup

Ollama is the local-first default and uses `http://localhost:11434` unless changed in Settings. Cloud providers require their API key to be enabled in Settings before inline completion runs.

Recommended low-latency setup:

1. Use a fast local Ollama coding model for everyday completion.
2. Keep inline completion delay between `250ms` and `450ms`.
3. Keep max completion tokens small for typing-time suggestions.
4. Use manual trigger for larger agentic completions.

## Troubleshooting

- If no ghost text appears, check the completion status in the status bar and the AI settings panel.
- If a cloud provider is selected, confirm that its API key toggle is enabled and the key is saved.
- If completion feels slow, use Ollama/local models or reduce the inline completion token limit.
- If `Tab` does not accept a suggestion, ensure ghost text is visible first; otherwise `Tab` follows normal editor behavior.
